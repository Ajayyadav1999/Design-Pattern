# Design Pattern

Design pattern are general reusable solutions to the common problems in software design. they are not full solutions i mean they are used as the templates or the best practices which we can adapt to design the software.


# Category of design patterns

 - Creational
 - Structural
 - Behavioral

# Creational 

Creational design patterns provides the various object creation mechanism to write the reusable and flexible code as well

   1. Factory method
   2. Prototype
   3. Abstract factory
   4. Builder
   5. Singleton


# Structural

Structural design patterns how we assemble the object and the classes in to the larger structure while keeping the code flexible and efficient.


# Behavioral

Behavioral design patterns means these are concerned with the algorithms and assignment of the responsibilities of the objects.
